# DocKit Modern Theme System

## Overview

DocKit uses a **Single CSS File** as the source of truth for all theme values. No JavaScript/TypeScript configuration needed - pure CSS for maximum simplicity.

## Architecture

```
src/assets/styles/index.css  ‚Üí Single source of truth (all theme defined here)
    ‚Üì
uno.config.ts               ‚Üí References CSS variables for utility classes
    ‚Üì
Vue Components              ‚Üí Use tokens via classes or CSS variables
```

## Single File Structure

### **src/assets/styles/index.css** (Only Theme File)

All theme values are defined here:

- Light mode CSS variables in `:root`
- Dark mode CSS variables in `[theme='dark']`
- Base Tailwind directives
- Global styles

```css
:root {
  --background: 0 0% 100%;
  --foreground: 0 0% 3.9%;
  --primary: 144 52% 45%; /* #36ad6a */
  --destructive: 341 73% 47%; /* #cd2158 */
  /* ... all other tokens */
}

[theme='dark'] {
  --background: 0 0% 3.9%;
  --foreground: 0 0% 98%;
  /* ... dark mode overrides */
}
```

## Using the Theme

### In Vue Components (Recommended)

Use utility classes generated by UnoCSS:

```vue
<div class="bg-background text-foreground border-border">
  <button class="bg-primary text-primary-foreground">Click me</button>
</div>
```

### In Scoped Styles

Use CSS variables directly (always wrap with `hsl()`):

```vue
<style scoped>
.custom-element {
  background: hsl(var(--background));
  color: hsl(var(--foreground));
  border: 1px solid hsl(var(--border));
}

.primary-button {
  background: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
}
</style>
```

## Available Tokens

All defined in [src/assets/styles/index.css](../src/assets/styles/index.css):

### Colors

- `--background` / `--foreground` - Base colors
- `--primary` / `--primary-foreground` - Brand primary (#36ad6a)
- `--secondary` / `--secondary-foreground` - Secondary actions
- `--destructive` / `--destructive-foreground` - Danger (#cd2158)
- `--muted` / `--muted-foreground` - Subtle text/backgrounds
- `--accent` / `--accent-foreground` - Hover states
- `--border` - Borders and dividers
- `--input` - Form inputs
- `--ring` - Focus rings
- `--card` / `--card-foreground` - Card containers
- `--popover` / `--popover-foreground` - Dropdown menus

### Spacing

- `--tool-bar-height` - Toolbar height (40px)
- `--radius` - Border radius (0.5rem)

## Theme Customization

**To change any theme value:**

1. Edit `src/assets/styles/index.css`
2. That's it! Changes apply everywhere automatically

Example - changing primary color:

```css
:root {
  --primary: 200 90% 50%; /* New blue color */
}
```

## Migration from Legacy

Components using old variables still work temporarily, but should migrate:

```css
/* Old (deprecated) */
background: var(--bg-color);
color: var(--text-color);

/* New (preferred) */
background: hsl(var(--background));
color: hsl(var(--foreground));
```

Or use utility classes:

```vue
<div class="bg-background text-foreground">
```
